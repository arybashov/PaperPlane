# PaperPlane

Браузерный симулятор маленького планера/бумажного самолета. Сейчас цель проекта - не красивая аркада, а проверяемая инженерная модель полета для небольших моделей с размахом до нескольких метров.

Текущий базовый объект - приближение к `BOO Slope Glider 800 mm`: 800 мм размах, V-образное оперение, низкие числа Рейнольдса, расчетный центр масс и настраиваемая геометрия.

## Что сейчас моделируется

- Жесткое тело с поступательным и угловым движением.
- Разнесенные аэродинамические поверхности: крыло, горизонтальное/V-оперение, киль.
- Локальный поток для каждой поверхности с учетом ветра, турбулентности, термиков и вращения модели.
- Табличная поляра профиля `E387 UIUC Re 200k` для `CL`, `CD`, `Cm`.
- Индуктивное сопротивление считается отдельно от профильного `CD`.
- Центр масс может быть ручным или расчетным по массам деталей.
- Конструктор геометрии меняет форму модели и физические параметры интерактивно.
- Оптимизация подбирает ручной ЦТ и угол хвоста для более ровного планирования.

Важно: это не CFD и не X-Plane. Модель нужна для быстрой отладки физики и параметров маленького планера.

## Текущий BOO-пресет

Пресет `BOO Slope Glider 800 mm` - это рабочая расчетная база, а не точная копия заводской модели. Геометрия и баланс подобраны как стартовая точка для отладки аэродинамики.

| Параметр | Значение |
| --- | --- |
| Размах крыла | `0.80 m` |
| Длина фюзеляжа | `0.485 m` |
| Масса | `170 g` |
| Площадь крыла | `7.8 dm2` |
| Профиль | `E387 UIUC Re 200k` |
| Оперение | `V-tail`, `40 deg` |
| Угол крыла | `0 deg` |
| Угол хвоста | `-0.5 deg` |
| Расчетный ЦТ | `~38.9 mm` от передней кромки крыла |
| ЦТ по САХ | `~39.5% MAC` |

Массы деталей в дефолте:

| Деталь | Масса | Плечо от носа |
| --- | ---: | ---: |
| Носовой груз/начинка | `47 g` | `0.058 m` |
| Крыло | `60 g` | центр площади крыла |
| Фюзеляж | `45 g` | середина фюзеляжа |
| V-оперение | `18 g` | центр площади хвоста |
| Киль | `0 g` | отключен для V-tail |

## Как считается центр масс

В расчетном режиме ЦТ считается по массам деталей:

```text
x_cg = sum(m_i * x_i) / sum(m_i)
```

`x_i` считается от носа самолета. Для крыла и хвоста берется центр площади соответствующей поверхности, для фюзеляжа - середина длины, для носовой массы - отдельный параметр `noseMassX`.

Если сумма масс деталей меньше общей массы, остаток считается полезной нагрузкой в точке `noseMassX`. Если сумма масс деталей больше общей массы, массы деталей масштабируются до общей массы, чтобы расчет ЦТ оставался согласованным.

В ручном режиме используется:

```text
cgX = wingLE + cgFromWingLE
```

То есть ручной ЦТ задается от передней кромки крыла, но в физику передается абсолютная координата от носа.

## Конструктор

Нижняя панель отвечает за модель:

- пресет модели;
- профиль крыла;
- общая масса;
- режим ЦТ: расчетный или ручной;
- массы деталей;
- размах, хорды и стреловидность крыла;
- положение крыла, хвоста и киля от носа;
- размеры оперения;
- тип хвоста: обычный или V-образный;
- углы крыла, хвоста, киля и V-tail.

Нос - нулевая точка координат. Крыло, хвостовые поверхности, киль и ЦТ считаются от него, чтобы не смешивать разные системы отсчета.

## Ветер и турбулентность

Ветер, турбулентность и термики разделены:

- ветер - постоянная скорость воздушной массы;
- турбулентность - локальная знакопеременная скорость воздуха, не скрытая подъемная сила;
- термики/восходящие потоки - устойчивый вертикальный поток воздуха.

В физике поверхность видит относительный поток:

```text
V_rel = V_surface - V_air
```

где `V_air` складывается из ветра, турбулентности и термиков.

## Структура файлов

| Файл | Назначение |
| --- | --- |
| `index.html` | разметка интерфейса |
| `style.css` | внешний вид сцены, боковой панели и конструктора |
| `app.js` | UI, параметры, пресеты, оптимизация, синхронизация конструктора |

Основной рабочий файл для аэродинамики - `physics.js`. `app.js` только собирает параметры из интерфейса и передает их в физику.

## Источники данных

- `E387 UIUC Re 200k` взят как табличная низкорейнольдсовая поляра из UIUC Low-Speed Airfoil Tests.
- BOO-параметры используются как приближение к Robbe BOO Slope Glider 800 mm.

Профиль E387 не подтвержден как реальный профиль BOO. Он выбран как доступная проверяемая поляра для отладки поведения маленького планера.

## Ограничения

- Нет полноценной нестационарной аэродинамики.
- Нет гибкости крыла и фюзеляжа.
- Нет интерполяции поляр по разным `Re`.
- Срыв и большие углы атаки приближенные.
- BOO-пресет пока не валидирован по реальным трекам полета.
